## Deploy: push to the repo

The repo already holds the old monolith `CLAUDE.md`, your `commands/` folder, and
`index.html` (the branded Decap login master). Unzipping this bible over the repo
**overwrites** `CLAUDE.md` with the lean core and **adds** everything else. Your four
existing command files and `index.html` are left untouched.

```bash
cd ~/Sites/claude-global-config

# copy the bible contents into the repo root (macOS example; the -R gets subfolders)
cp -R /path/to/claude-global-config-bible/. .

git add -A
git commit -m "Rebuild: lean CLAUDE.md + specs + skills + ask-the-board + qa command"
git push origin main
```

Remote: `git@github.com:livewebstudios/claude-global-config.git`, branch `main`.

## Install per machine (after cloning / pulling)

### Mac (GoldieAir)
```bash
chmod +x ~/Sites/claude-global-config/install-mac.command
~/Sites/claude-global-config/install-mac.command
```
Symlinks `CLAUDE.md`, `specs/`, `commands/`, and each of your skills individually into
`~/.claude/`, so any borrowed skills already in `~/.claude/skills/` survive.

### Windows 11 Pro
```bat
%USERPROFILE%\Sites\claude-global-config\install-windows.bat
```
Copies the same set into `%USERPROFILE%\.claude\` (no symlinks, per your Windows policy).
Confirm the `REPO` path at the top of the .bat matches where your Windows clone actually
lives. Your existing `synccmd.bat` still handles the reverse `.claude` -> repo commit.

### Both machines: borrowed skills
```
npx skills add pbakaus/impeccable
```
See `REQUIRED-SKILLS.md`. `ui-ux-pro-max` install command is still TBD, record it there
once confirmed.

## Em dash note

The em dash ban is a **client-facing rule**, not a repo-wide one. It exists because a
client flagged the em dash as an AI tell, so it applies to what clients and the public
read: emails, letters, proposals, web copy, blog, social. It does NOT apply to internal
config, specs, skills, or code, nobody grades you on those.

For reference: the authored config files (`CLAUDE.md`, the specs, the personas) were
scrubbed em-dash-clean anyway as belt-and-suspenders. `jon_voice_master.md` intentionally
keeps its em dashes (it documents the ban and quotes real pre-ban samples), and your
pre-existing standalone skills plus `clients.md` still carry theirs, verbatim. None of
that matters under the client-facing scope. A repo-wide scan flagging those is expected
and fine.
